## First Run & Configuration

The scanner uses an interactive setup wizard. Run it once to configure everything:

```bash
python launch.py
```

The wizard walks you through:

<CardGrid>
  <Card title="Platform Selection" icon="laptop">
    Choose Emulator (MuMu, LDPlayer, etc.), Desktop Client, or Real Device.
  </Card>
  <Card title="ADB Connection" icon="setting">
    Enter a custom IP/port. (Auto-fill ports coming soon.)
  </Card>
  <Card title="Scan Targets" icon="list-format">
    Choose which inventories to scan: Equipment, Items, Students, Currencies.
  </Card>
  <Card title="Performance Tuning" icon="rocket">
    Adjust wait-time multipliers for slower devices or emulators.
  </Card>
  <Card title="Data Sync" icon="down-caret">
    Enable or disable automatic downloads of the latest game asset data.
  </Card>
</CardGrid>

<Aside type="tip">
  Your choices are saved to `config/settings.json` and `config/screen_config.json`. You won't need to run the wizard again unless you want to change settings.
</Aside>

## Running the Scanner

<Tabs>
  <TabItem label="Standard Run">
    Uses your saved settings:
    ```bash
    python launch.py
    ```
  </TabItem>
  <TabItem label="Edit Configuration">
    Re-runs the wizard with your current values as defaults:
    ```bash
    python launch.py -e
    # or
    python launch.py --edit
    ```
  </TabItem>
  <TabItem label="Offline Mode">
    Skips network sync and runs directly:
    ```bash
    python main.py --offline
    ```
  </TabItem>
</Tabs>

Once started, the scanner automatically navigates the game, captures screenshots, extracts data via **RapidOCR**, and saves results to the `output/` directory.

## Output Files

After a successful scan, check the `output/` folder:

| File | Description |
|---|---|
| `owned/scanned_counts.json` | Raw item & equipment counts |
| `owned/scanned_students.json` | Raw student stats |
| `owned/scanned_currencies.json` | AP, Credits, Pyroxene |
| `equipment_final_values.json` | Processed & mapped equipment data |
| `items_final_values.json` | Processed & mapped items data |
| `students_final_values.json` | Processed & mapped student data |

## Optional: Justin Planner Export

### Convert to Justin Planner Format

<Steps>

1. Run the converter script:

   ```bash
   python convert_justin_planner.py
   ```

   This generates `output/converted_to_justin_planner.json`.

</Steps>

### Merge into Your Existing Justin Planner Data

<Steps>

1. Save your Justin Planner export as `justin_data.json`.

2. Place it at:
   ```
   input/justin_data.json
   ```

3. Run the merger script:
   ```bash
   python merger_justin_planner.py
   ```

4. Import the generated `output/justin_data_final.json` into Justin Planner.

</Steps>
